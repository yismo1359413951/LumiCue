## 🇨🇳 写在前面

我是一个 vibe coding 的初学者，技术很浅，这个小东西基本是我一边问 AI、一边照着学，慢慢磨出来的练手作业。

起因很简单：我需要一个录屏时观众看不见的提词器。找到了 Snapzy—— 一个很棒的开源截屏录屏工具，里面刚好有个提词器。但它还带着菜单栏、截图、录屏、标注…… 一整套东西，我全都不需要。于是我问自己：如果把这些全砍掉，只留下提词器，把它装进一个毛玻璃泡泡里——漂在屏幕上，只有我能看见，别人都看不见——会是什么样子？

这就是 LumiCue。启动直接弹提词器窗口，关掉就退出。没有菜单栏，没有状态图标，没有多余功能——就是一个安安静静、有点好看的小窗，帮你边念边滚字幕。

水平有限，代码里一定有很多不成熟的地方，欢迎指正、欢迎 PR，我也在学。

---

## ✨ 为什么叫「靓相」

"Lumi" 来自 luminous——发光、微微亮着。"Cue" 是你的提词、你的下一句、你沉默的帮手。

靓相，意思是"美丽的在场"——它就陪在你屏幕边上，观众看不见，但你因为它在，变得更自然、更从容。

不是正经工具。是一点小小的陪伴——你信赖它，它不打扰你，默默让你在镜头前更好。

## 💜 它长什么样

- **毛玻璃**半透明底层——背后的内容隐约透出来，不挡死
- **彩虹流光描边**——靛蓝→紫→粉红渐变，缓缓流转
- **发光的进度条**，前端一颗脉冲小彗星
- **天色随念稿进度变化**——白天→黄昏→黑夜，陪你走完一段稿
- **12 种字色**，Tailwind 柔和配色——不刺眼，念久了不累

## 🧩 它有什么用

### 📺 天生隐形
窗口用了 `sharingType = .none`。你录屏、直播、截屏——观众永远看不到它。只有你能看到。

### 🫧 两种形态：念稿态 & 收起态
- **念稿态（大框）**：可拖拽调大小的主窗口，居中滚动文字 + 全部控制 + 进度轨 + 救场面板
- **收起态（胶囊）**：一条很窄很矮的横条，小字 2–3 行 + 迷你控制——拖到角落，不占画面

### 📝 粘贴即念
- `⌘V` 直接粘逐字稿——不用点任何按钮，当场显示
- 右键菜单导入 `.txt` 文件
- 内置编辑器自动删所有隐形空格（零宽空格、全角空格、NBSP……全清掉），保留标点和换行

### 🎵 播控
- 速度 0.1×（慢爬）到 2×（快滚）
- 12 种系统字体 + 3 档窗口预设（小/中/大）+ 手动字号缩放（A-/A+）
- 空格键暂停/继续
- 拖进度条跳到稿子里任意位置
- `↑` 回退一句

### 🆘 救场面板
暂停时自动弹出，念稿态和收起态都有：
- **↺ 重念这句**——当前句从头再来
- **← 退 N 句**——退 1 句、2 句、3 句…… 累加计数
- **▶ 继续**——恢复滚动

### 🎤 语音跟随（实验功能）
内置离线语音识别（中文）——数据不出你的电脑。你说到哪，字幕跟到哪。还比较粗糙，但能用。

### 🌐 中英一键切
控制条一个按钮切换所有按钮语言。按钮显示的是**对面的语言**——中文界面显示"EN"，英文界面显示"中"。

## 🔧 构建

> 需要 **macOS 13.0** 或更高版本。

```bash
git clone <本仓库地址>
cd LumiCue
open Snapzy.xcodeproj
```

1. 选择 **Snapzy** scheme
2. 编译（⌘B）并运行（⌘R）
3. 提词器窗口立即弹出——就这样

> Xcode 项目内部保留了 `Snapzy` 这个 bundle 名以保证数据兼容。所有用户界面显示为 **LumiCue**。

## 👻 已知问题：字幕重影

滚动时，文字可能有轻微的重影/双像——高速滚动或低刷新率外接显示器上更明显。

大概率是 `CATextLayer` 合成和毛玻璃 `NSVisualEffectView` 底层在渲染管线里的交互问题。文字层快速移动时似乎会留下微弱的拖尾。

**非常欢迎 PR！** 一些值得尝试的方向：
- 降低毛玻璃底层更新频率
- 文字渲染切到 `NSTextField` 或带 `drawsAsynchronously` 的 `CALayer`
- 帧同步的 `CATransaction` flush
- `CVDisplayLink` 对齐刷新的滚动
