## 一、头号结论（国内外完全一致）

**最痛的不是技术，是"被看出在念稿 / 眼神飘 / 显得不真诚"。** 国内外所有来源都把它列第一。其次是"语音跟随做不稳""卡壳/重录的心理压力"。商业上最大怒点是"订阅套路 + 核心功能付费墙 + 语音识别根本不工作"。

## 四、差异化定位（开源·国内外·大气）
1. **开源 + 免费/买断** —— 砍掉"订阅套路/付费墙"最大怒点，国内外通吃。
2. **本地语音跟随**（设备端识别不上传）—— 隐私+不卡+不限时，治痛点2/3，技术护城河。
3. **治眼神飘** —— 短句喂词 + 贴镜头光轴，从源头让人不像在读。
4. **设计大气**：克制高级（参考 Linear / Raycast）—— 去掉幼稚元素（小猫/烟花/火花连击太儿童）。

---

## 五、"好玩/小巧思"——成熟版（替换幼稚的）
1. **语音跟随的丝滑本身就是爽**：字跟着嘴走、像被托着，不用追字幕。
2. **呼吸节奏引导**：该停顿/换气处文字极轻微"呼吸"，高级又帮你说自然。
3. **录后极简数据卡**（像运动手环）：流畅度/卡壳次数/平均语速，给进步感、不幼稚。
4. **卡壳兜底**：卡住稳稳停、给下一句关键词（不是整句），救场不慌。

## 六、下一步功能优先级（你从这选）
- 🔴 A. **本地语音跟随**：最硬护城河，治"速度对不上+卡壳"。技术：macOS Speech 框架本地中英识别。
- 🔴 B. **卡壳救场**：已做完（空格暂停+重念这句/退N句/继续），待你界面验。
- 🟡 C. **短句喂词排版**：治眼神飘。
- 🟡 D. **念过飘散 + 字体可选**：收尾。
- ⚪ E. **换成熟设计语言**：去掉小猫/烟花，改克制高级风。

**我的建议顺序：B（验收）→ A（护城河）→ C → E → D。** 你定。
