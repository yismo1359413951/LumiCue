## 这是什么？

LumiCue 是一个 **macOS 悬浮提词器**。它浮在你屏幕最上面，你直播/录屏/开会时照着念——但因为用了 `sharingType = .none`，**观众和录屏软件完全看不到它**。只有你能看到。

从开源截屏录屏神器 [Snapzy](https://github.com/duongductrong/Snapzy) 魔改而来，砍掉了所有截屏/录屏/标注功能，变成一件纯粹的工具：**帮你自然地看着镜头念稿，观众毫无察觉。**

## 功能

### 📺 隐形提词
- 窗口对**录屏、直播、截图完全不可见**（`sharingType = .none`）
- 你看着念，观众完全不知道你在看稿

### 🫧 双模式：念稿态 & 收起态
- **念稿态（大框）**：可拖拽改大小的主窗口，居中滚动文字 + 控制条 + 进度轨道 + 救场面板
- **收起态（胶囊）**：一条很窄的横条（2–3 行小字 + 迷你控制），拖到镜头下面不占地方

### 🎨 好看好玩的 UI
- **毛玻璃**底层 + 随念稿进度变化的天色（白天→黄昏→黑夜）
- **彩虹流光描边**（靛蓝→紫→粉红渐变流动）
- **发光进度条** + 脉冲彗星头
- **12 种字色**来自 Tailwind 配色：白、琥珀、橙、珊瑚、粉、薰衣草、靛蓝、天蓝、青、薄荷、柠绿、米白

### 📝 智能稿本输入
- **⌘V 直接粘贴**——不用点任何按钮，粘贴即显示
- **右键菜单导入 .txt** 逐字稿
- **内置编辑器**自动去空格（删所有空白符，保留标点和换行）
- 处理 NBSP、零宽空格、全角空格等所有隐形 Unicode 空白

### 🎵 播放控制
- **速度可调**：0.1× 到 2×（从慢爬到快滚）
- **字体可配**：12 种系统字体 + 6 档预设（小/中/大）+ 手动缩放（A-/A+）
- **空格键暂停/继续**
- **进度条拖拽定位**：拖到哪里字幕跳到哪
- **↑ 回退一句**

### 🆘 救场面板（暂停时弹出）
- **↺ 重念这句**——当前句从头再念
- **← 退 N 句**——退 1 句、2 句、3 句……（累加计数）
- **▶ 继续**——恢复滚动

### 🎤 语音跟随（实验功能）
- 内置离线语音识别（中文，本地处理，不上传数据）
- 你说到哪，字幕跟到哪——实时同步

### 🌐 中英文界面
- 控制条「EN ⇄ 中」一键切换所有按钮语言（默认中文）
- 按钮显示的是**对面的语言**：中文界面显示"EN"，英文界面显示"中"

### ⌨️ 快捷键
- `空格` — 暂停 / 播放
- `↑` — 回退一句
- `↓` — 前进一句
- `⌘V` — 从剪贴板粘贴逐字稿

### 🖥️ macOS 原生
- SwiftUI + AppKit + Core Animation 构建
- 深色外观 + 毛玻璃
- 拖窗口任意位置即可移动
- 右下角拖拽手柄改大小
- 支持全屏辅助窗口

## 安装

> 需要 **macOS 13.0** 或更高版本。

### 从源码编译

```bash
git clone https://github.com/yismo1359413951/LumiCue.git
cd LumiCue
open Snapzy.xcodeproj
```

1. 选择 **Snapzy** scheme
2. 编译（⌘B）并运行（⌘R）
3. 提词器窗口即刻弹出——没有菜单栏图标，没有状态栏

> **注意**：Xcode 项目内部保留了 bundle identifier `Snapzy` 以兼容数据迁移。所有用户界面显示为 **LumiCue**。

## 已知问题

### 👻 字幕重影 Bug

在提词器滚动时，部分用户可能会注意到文字有**轻微的重影/双像**——尤其在高速滚动或低刷新率外接显示器上更明显。

这大概率是 `CATextLayer` 合成与 `NSVisualEffectView` 毛玻璃背景在当前渲染管线中的交互问题。文字层在快速位置更新时可能留下微弱的拖尾。

**如果你有修复思路，非常欢迎提 PR！** 一些值得探索的方向：
- 降低 `NSVisualEffectView` 背景更新频率
- 将文字渲染切换到 `NSTextField` 或带 `drawsAsynchronously` 的 `CALayer`
- 增加帧同步的 `CATransaction` flush
- 尝试用 `CVDisplayLink` 做 vsync 对齐的滚动
