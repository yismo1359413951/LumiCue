## 关于

LumiCue 是一个小小的 macOS 提词器，起因特别简单：我录视频需要有个稿子滚在眼前，但不想让观众看见。我找到了 Snapzy——一个很棒的开源截屏录屏工具，里面刚好带了个提词器。但它还有菜单栏、截图、录屏、标注…… 一整套餐具，我只要那一只碗。于是我把其他全拆了，只留下提词器，装进一个毛玻璃泡泡里。

打开就是提词器，关掉就退出。没有菜单栏，没有状态图标——就是一块安安静静、有点好看的玻璃，帮你看着镜头念稿。

中文名叫「灵光提示」——念稿卡壳时，那句词刚好亮在眼前，像脑子里灵光一闪。

---

## ✨ 为什么叫这个名字

**Lumi** 来自 luminous——微微发着光。**Cue** 是你的提词、你的下一句。

不是正经工具。是屏幕上一小块陪伴——你信赖它，它不打扰你，默默让你在镜头前更好。

## 💜 它长什么样

- **毛玻璃**半透明底层——背后内容隐约透出，不挡死视野
- **彩虹流光描边**——靛蓝→紫→粉红，缓缓流转
- **发光的进度条**，前端一颗脉冲小彗星
- **天色随念稿进度变化**——白天→黄昏→黑夜，陪你走完一段稿
- **12 种 Tailwind 字色**——柔和、清楚、念久了不累

## 🧩 它能做什么

### 📺 天生隐形
窗口用了 `sharingType = .none`。你录屏、直播、截屏——观众**永远**看不见它。只有你看得到。

### 🫧 两种形态：念稿态 & 收起态
- **念稿态（大框）**：可拖拽调整大小，居中滚动文字 + 全部控制 + 进度轨 + 救场面板
- **收起态（胶囊）**：一条很窄很矮的横条，小字 2–3 行 + 迷你控制——拖到镜头下面，完全不占画面

### 📝 粘贴即念
- `⌘V` 直接粘逐字稿，当场显示
- 右键菜单导入 `.txt` 文件
- 内置编辑器自动清掉所有隐形空格（零宽空格、全角空格、NBSP……全删干净），保留标点和换行

### 🎵 播控
- 速度 0.1× 到 2×
- 12 种系统字体 + 3 档窗口预设（小/中/大）+ 手动字号缩放（A-/A+）
- 空格键暂停/继续
- 拖进度条跳到稿子里任意位置
- `↑` 回退一句

### 🆘 救场面板
暂停时自动弹出，念稿态和收起态都有：
- **↺ 重念这句**——当前句从头再来
- **← 退 N 句**——退 1 句、2 句…… 累加计数
- **▶ 继续**——恢复滚动

### 🎤 语音跟随（实验功能）
内置离线语音识别（中文）——数据不出你的电脑。你开口念，字幕跟着走。

### 🌐 中英一键切
控制条一个按钮切换所有按钮语言。按钮显示的是**对面的语言**——中文界面显示"EN"，英文界面显示"中"。

## 🔧 构建

> 需要 **macOS 13.0** 或更高版本。

```bash
git clone <仓库地址>
cd LumiCue
open Snapzy.xcodeproj
```

1. 选择 **Snapzy** scheme
2. 编译（⌘B）并运行（⌘R）
3. 提词器窗口立即弹出——就这样

> Xcode 项目内部保留了 `Snapzy` 这个 bundle 名以保证数据兼容。所有用户界面显示为 **LumiCue**。

## 👻 已知问题：字幕重影

滚动时，文字可能有轻微的重影/双像——高速滚动或低刷新率外接显示器上更明显。

大概率是 `CATextLayer` 合成和毛玻璃 `NSVisualEffectView` 底层在渲染管线里的交互问题，文字层快速移动时似乎会留下微弱拖尾。

**非常欢迎 PR！** 一些值得尝试的方向：
- 降低毛玻璃底层更新频率
- 文字渲染切到 `NSTextField` 或带 `drawsAsynchronously` 的 `CALayer`
- 帧同步的 `CATransaction` flush
- `CVDisplayLink` 对齐刷新的滚动
