# 靓相 Shotlit · 交接给新对话（必读）

> 换新对话先读这份 + tasks/todo.md + 每日复盘.md。真相在文件里。

## 一句话目标
**让你出镜好看的录屏软件**：Screen Studio 级录屏 + 直播级实时美颜露脸 + 异形露脸框 + 隐形提词器。中文名「靓相」/ 英文 Shotlit。基座 = Snapzy（开源 CleanShot 替代品，BSD 协议可商用二开）。

## 当前进度（2026-06-23 第一天，全界面验证）
环境就绪 + 已做：露脸 bubble、异形框 8 形状、Metal 双边磨皮+美白、9 个 LUT 滤镜、人脸跟随(缩小居中+模糊背景)、镜像、整脸瘦脸、隐形提词器(可调)、单击选项菜单。

## 🔴 关键决策/事实（别推翻、别重踩）
1. **必须 Xcode 26.x**（Snapzy 用 SWIFT_DEFAULT_ACTOR_ISOLATION=MainActor=Swift6.2）。本机已装 26.3。16.4 死路。
2. **Metal 工具链已装**（Xcode 26 拆成可选组件：`xcodebuild -downloadComponent MetalToolchain`）。
3. **美颜路线 = 手搓**（用户定）：Vision 76 点关键点逐部位精细美颜。别用 Core Image 凑合(雾蒙蒙)。
4. **隐形提词器 = `NSWindow.sharingType = .none`**（录屏/直播隐形，本地可见，ScreenCaptureKit 对照验证过）。
5. **验证隐形/录屏类**：必须用 ScreenCaptureKit(真捕获)，不能用 screencapture 全屏(特例不遵守 sharingType)。

## 编译运行命令（在 snapzy/ 目录）
```bash
cd ~/Desktop/Shotlit/snapzy
xcodebuild -project Snapzy.xcodeproj -scheme Snapzy -configuration Debug \
  -derivedDataPath /tmp/snapzy-build \
  CODE_SIGNING_ALLOWED=NO CODE_SIGNING_REQUIRED=NO \
  INFOPLIST_KEY_NSCameraUsageDescription="Shotlit needs the camera to show your face bubble. 靓相需要摄像头来显示你的露脸画面。" \
  build
# 产物未签名→摄像头被TCC拒,必须 ad-hoc 签名:
codesign --force --deep --sign - "/tmp/snapzy-build/Build/Products/Debug/Snapzy Debug.app"
open "/tmp/snapzy-build/Build/Products/Debug/Snapzy Debug.app"
# ⚠️每次重编 ad-hoc hash变,摄像头权限会重问(根治:Xcode登录AppleID用开发签名,未做)
```
