## 2. 🔴🔴 三个致命坑（不读必重踩）
### 坑1：摄像头会被"还原器"偷偷打开
`Snapzy/App/SnapzyApp.swift` 里 `bubble.show()`（约126行）一启动就开摄像头。有个进程（编辑器/linter）**会在编译瞬间把我的注释还原回去**，导致每次都开摄像头/崩溃。
**解法（编译前必做）**：perl 注释那三行 → `chmod 444` 锁文件 → 编译 → `chmod 644`。见第4节命令。

### 坑2：语音跟随在这台机器做不了（苹果方案）
用户是 **Intel Mac**（无 ANE，苹果本地中文识别不支持）+ **全局 VPN**（挡苹果在线语音服务器）→ 苹果 `SFSpeechRecognizer` 跑不通。
- 语音代码（`VoiceFollower` 类、🎤逻辑）保留在 TeleprompterWindow.swift，但🎤按钮已撤出控制条。
- 真要做语音跟随 → 用**本地 Whisper.cpp**（离线、不靠苹果芯片/网络），是大工程，UI 做好后再单独啃。

### 坑3：编译"成功"≠界面更新
增量编译/旧实例残留会让她看到旧界面。改完务必：`pkill -9 -f Snapzy` → 重编译 → 重签名 → `open`。必要时 `clean build`。

## 4. 编译 + 运行命令（照抄）
```bash
cd ~/Desktop/Shotlit/snapzy
export PATH="/usr/local/bin:$PATH"
pkill -9 -f Snapzy 2>/dev/null
# 坑1: 注释摄像头 + 锁文件防还原
perl -i -pe 's{^(\s*)let bubble = CameraBubbleWindow\(\)}{$1// let bubble = CameraBubbleWindow()}; s{^(\s*)bubble\.show\(\)}{$1// bubble.show()}; s{^(\s*)cameraBubble = bubble}{$1// cameraBubble = bubble}' Snapzy/App/SnapzyApp.swift
chmod 444 Snapzy/App/SnapzyApp.swift
xcodebuild -project Snapzy.xcodeproj -scheme Snapzy -configuration Debug -derivedDataPath /tmp/snapzy-build \
  CODE_SIGNING_ALLOWED=NO CODE_SIGNING_REQUIRED=NO \
  INFOPLIST_KEY_NSMicrophoneUsageDescription="mic" INFOPLIST_KEY_NSSpeechRecognitionUsageDescription="sp" \
  build 2>&1 | grep -E "BUILD SUCCEEDED|BUILD FAILED|error:"
chmod 644 Snapzy/App/SnapzyApp.swift
APP="/tmp/snapzy-build/Build/Products/Debug/Snapzy Debug.app"
codesign --force --deep --sign - --entitlements Snapzy/Snapzy.entitlements "$APP"
open "$APP"
```
注意：**不要**加 `INFOPLIST_KEY_NSCameraUsageDescription`（不声明摄像头，系统就不可能弹摄像头框）。
