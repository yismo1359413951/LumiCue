## 🔧 编译命令（必前台确认 BUILD SUCCEEDED，别拿 exit0 当成功！）
```
cd ~/Desktop/Shotlit/snapzy
export PATH="/usr/local/bin:$PATH"
pkill -x Snapzy 2>/dev/null
xcodebuild -project Snapzy.xcodeproj -scheme Snapzy -configuration Debug \
  -derivedDataPath /tmp/snapzy-build CODE_SIGNING_ALLOWED=NO CODE_SIGNING_REQUIRED=NO \
  INFOPLIST_KEY_NSCameraUsageDescription="Shotlit camera" build 2>&1 | grep -E "BUILD SUCCEEDED|error:"
APP="/tmp/snapzy-build/Build/Products/Debug/Snapzy Debug.app"
codesign --force --deep --sign - "$APP"; open "$APP"
```

## 🕳️ 血泪坑（务必记住）
1. **background 编译 exit0 ≠ BUILD SUCCEEDED** → 必 grep "BUILD SUCCEEDED" + pgrep 确认 app 起来；关键修复用**前台编译**当场确认。（误报过两次，害用户一直跑旧版）
2. **ad-hoc 签名每次变 → 摄像头 TCC 权限重置 → screencapture 截 bubble 是黑的** → 让用户截图发你。
3. NSLog 进系统统一日志、不进 stderr → 用 `log show --predicate 'process=="Snapzy"'` 抓。
4. **隐形靠 `sharingType=.none`，不是视觉透明**；提词器背景要么毛玻璃(后面模糊)、要么不透明，**别用半透明**(透出后面叠字)。文字**别加硬描边**(用户否决)。
5. 用户 Intel x86_64：第三方库要 x86_64（GpuPixel 是从源码自编的 x86_64）。
6. **生成的图/截图必须 open + 拷桌面给用户看**（Read 的图她看不到）。
7. 设计/做产品前**先用 brainstorming skill 想透**，别拍脑袋瞎做（瞎做翻车一整天的教训）。
